## Setup — build the database

All material EPD data, section properties, deck properties, and floor build-up configurations are stored in an Excel master file and imported into a SQLite database before any optimisation is run.

**This step is required only once** (or whenever the Excel data is updated).

1. Ensure `210526_Datenbankdefinition.xlsx` is in the same folder as the scripts.
2. Run:

```bash
python Convert_Excel_to_DB.py
```

This creates `database_270426.db` in the same folder. All subsequent scripts read from this file.

## Reproducing results — step by step

### Step 1 — Run the optimisations

Each of the three run scripts (`simple_span_composite.py`, `two_span_composite.py`, `three_span_composite.py`) loops over all combinations of use-case, design criterion, and slab type, and saves results to CSV.

Open each file and check the **USER INPUTS** block at the top before running. The key parameters are:

| Parameter | Description | Options |
|---|---|---|
| `use_case` | Occupancy type and loading | `"office"`, `"residential"`, `"retail"`, `"industrial"` |
| `criteria` | Design criteria envelope | `["ENV"]`, `["ULS"]`, `["SLS1"]`, `["FIRE"]` |
| `lengths` | Spans to evaluate [m] | e.g. `[3, 4, 5, 6, 7, 8, 9, 10]` |
| `max_iter` | Optimisation iterations | `100` (increase for better accuracy, slower) |

Run each script for **every combination** of use-case and criterion to generate the full set of results CSVs. For example:

```bash
python simple_span_composite.py    # produces results_<usecase>_<criterion>.csv
python two_span_composite.py       # produces results_<usecase>_2span_<criterion>.csv
python three_span_composite.py     # produces results_<usecase>_3span_<criterion>.csv
```

> **Note:** Each run appends to the CSV file. Delete or rename existing CSVs before a fresh run to avoid duplicating data.

The full set of results used in the report was generated with:
- Use-cases: `office`, `residential`, `retail`, `industrial`
- Criteria: `ENV`, `ULS`, `SLS1`, `FIRE`
- Spans: 3–10 m

### Step 2 — Verify against manufacturer tables

```bash
python verify_mfr_tables.py
```

Checks every optimised composite slab configuration against the manufacturer's published load-span tables (ComFlor and Kingspan Multideck manuals). Results are saved to `mfr_table_verification.csv`. The script prints a PASS/FAIL summary and saves the CSV for use in the verification figure.

Key assumptions hard-coded in the file:
- Office load: 5.00 kN/m² total applied
- Fire resistance: R60
- Concrete: Normal Weight Concrete (NWC)
- ComFlor: Eurocode Bar Fire Method tables; beam width 152 mm
- Kingspan: British Standard tables; 6.0 kN/m² column (conservative)

### Step 3 — Generate summary figures

```bash
python plot_composite_summary.py
```

Reads the results CSVs and produces six publication figures saved as PNGs in the project folder:

| Output file | Contents |
|---|---|
| `fig_composite_span_configs.png` | Office composite GWP vs span (1-, 2-, 3-span) |
| `fig_composite_vs_rc_occupancies.png` | Composite vs RC solid across all occupancies (struct + total GWP) |
| `fig_mfr_verification.png` | Manufacturer table verification dot plot |
| `fig_criteria_comparison_SS.png` | ENV vs ULS vs SLS vs FIRE — single span |
| `fig_criteria_comparison_2span.png` | ENV vs ULS vs SLS vs FIRE — two-span |
| `fig_criteria_comparison_3span.png` | ENV vs ULS vs SLS vs FIRE — three-span |

### Step 4 — GWP component breakdown

```bash
python gwp_breakdown_all.py
```

Edit the `CSV_FILE` variable at the top to select which results file to analyse (e.g. `"results_office_ENV.csv"`). Produces stacked-bar breakdown figures for all five slab types, split by concrete / deck / mesh / trough bars, with and without floor build-up. Figures are saved to `plots/<use_case>/GWP breakdown/`.

### Step 5 — Building envelope / storey count

```bash
python building_envelope.py
```

Computes the number of storeys achievable in a fixed building height for each slab type and span. Building parameters (height, clear height, ceiling void) are set in the `USE_CASES` dictionary at the top of the file. Outputs:
- Printed tables to the console
- `building_envelope_results.tex` — LaTeX tables ready to `\input{}` in a report
- Three PNG figures saved to the project folder

### Step 6 — Supplementary checks (optional)

**Vibration check (standalone):**
```bash
python full_vibration_check.py
```
Runs a single floor bay through the SCI P354 §7 simplified vibration method. Edit the geometry and loading in the USER INPUTS section at the top.

**Batch vibration check from CSVs:**
```bash
python verify_vibration.py
```
Reads the three optimisation CSVs and checks the slab-alone natural frequency (≥ 4 Hz office criterion) for every unique composite slab configuration.

**End anchorage comparison:**
```bash
python end_anchorage_comparison.py
```
Compares moment capacity and GWP with and without headed-stud end anchorage for all τᵤ composite decks in the database, per EN 1994-1-1 Cl 9.7.4.

## Script reference

### `struct_analysis.py`
The structural analysis engine. Contains all section classes and design check implementations:

- **Section classes:** `CompositeSlab`, `RectangularConcrete`, `RibbedConcrete`, `RectangularWood`, `RibWood`, `Decking`, `FloorStruc`
- **System classes:** `BeamSimpleSup`, `BeamTwoSpan`, `BeamContinuousSupEl`
- **Design checks per Eurocode:** ULS bending/shear, SLS deflection, vibration (SCI P354), fire (NCCI PN005 Bar Fire Method), longitudinal shear (partial connection / m-k)
- Not intended to be run directly — imported by all other scripts.

### `struct_optimization.py`
Optimisation routines. For each section type and span, iterates over depth/reinforcement combinations to find the minimum-GWP section that passes all active design checks. Key functions:
- `opt_comp_slab()` — composite slab optimiser (unpropped preferred, propped fallback)
- `get_optimized_section()` — RC and timber optimiser

### `plot_datasets.py`
Shared plotting utilities. Contains:
- `run_dataset()` — runs optimisation and saves results to CSV (called by the run scripts)
- `plot_from_csv()` — generates envelope band plots from a saved CSV
- `add_dataset_legend()` — adds the shared slab-type legend to figures

### `simple_span_composite.py` / `two_span_composite.py` / `three_span_composite.py`
Top-level run scripts. Each configures the analysis (use-case, criterion, spans) and calls `run_dataset()` for every slab type. Edit the **USER INPUTS** block and run directly.

### `verify_mfr_tables.py`
Standalone verification script. Contains manufacturer table data hard-coded from the ComFlor (Tata Steel) and Kingspan Multideck manuals:
- All ComFlor decks use the **Bar Fire Method** tables (EC, R60, NWC)
- Kingspan decks use the **standard load/span** tables at 6.0 kN/m² (double span)
- Results are interpolated linearly between tabulated depths

### `plot_composite_summary.py`
Post-processing figures script. All figures are saved as PNGs. No optimisation is run — reads pre-computed CSVs only. Can be re-run at any time after the optimisation CSVs exist.

### `gwp_breakdown_all.py`
GWP component breakdown bar charts. Set `CSV_FILE` at the top. Composite slab bars are split into: **Concrete / Steel deck / Mesh / Trough bars**. RC and timber bars show total structural GWP as a single block. A second figure adds the floor build-up layers on top.

### `building_envelope.py`
Storey-count analysis. Formula:
```
floor-to-floor = h_tot_m  +  clear_height  +  ceiling_void
n_storeys = building_height / floor-to-floor
```
`h_tot_m` (from the CSV) includes the structural depth plus all floor build-up layers above the slab. `n_storeys` is reported as a continuous number; the achievable integer count is `floor(n_storeys)`.

### `Convert_Excel_to_DB.py`
One-time database builder. Reads four sheets from the Excel master file (`products`, `material_prop`, `floor_struc_prop`, `sheeting_prop`) and writes them to a SQLite database. Run this once before any optimisation.

## Output files

| File pattern | Generated by | Description |
|---|---|---|
| `results_<usecase>_<criterion>.csv` | Run scripts | Optimisation results — one row per section × span |
| `results_<usecase>_<N>span_<criterion>.csv` | Run scripts | As above for multi-span configurations |
| `mfr_table_verification.csv` | `verify_mfr_tables.py` | PASS/FAIL against manufacturer tables |
| `building_envelope_results.tex` | `building_envelope.py` | LaTeX tables for storey-count results |
| `fig_*.png` | `plot_composite_summary.py`, `building_envelope.py` | Publication figures |
| `plots/<usecase>/GWP breakdown/*.png` | `gwp_breakdown_all.py` | GWP breakdown bar charts |
